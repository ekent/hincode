## ⏸️ MVP 限制

**已知限制 (将在后续版本改进):**

1. **只审查第一个文件**
   - 原因: MVP 简化
   - 计划: v0.2 支持多文件并行

2. **无配置文件支持**
   - 原因: 硬编码配置
   - 计划: v0.2 添加 `.hincode.toml`

3. **无 tree-sitter 集成**
   - 原因: MVP 只做基础功能
   - 计划: v0.2 集成 AST 解析

4. **无缓存机制**
   - 原因: 简化实现
   - 计划: v0.3 添加结果缓存

5. **无交互式修复**
   - 原因: MVP 聚焦审查
   - 计划: v0.4 添加交互模式

## 📚 使用统计

**命令行选项:**
```
-b, --base <BASE>      Base branch [default: main]
-o, --output <OUTPUT>  Output format [default: terminal]
-v, --verbose          Verbose output
```

**支持的语言:**
- Rust, Python, JavaScript, TypeScript
- Go, Java, C, C++

**输出格式:**
- Terminal (彩色)
- Markdown (PR 友好)
- JSON (机器可读)
